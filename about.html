VALORES & FILOSOFIA
01
PERFORMANCE É UMA FEATURE

Latência de API, tempo de resposta, uso de memória — não são detalhes, são requisitos. Código que funciona mas é lento é código incompleto. Performance é parte do produto, não bônus.

02
CÓDIGO LEGÍVEL É DOCUMENTAÇÃO

Código que precisa de comentário para ser entendido é código mal escrito. Nomes claros, funções pequenas, responsabilidades separadas — isso é respeito pelo próximo desenvolvedor (e pelo futuro eu).

03
AUTOMAÇÃO PARA TUDO QUE REPETE

Se você faz a mesma coisa mais de duas vezes, é hora de automatizar. Pipelines, scripts, bots — liberar tempo humano para trabalho que realmente exige criatividade e raciocínio.

04
SISTEMAS QUE ESCALAM

Arquitetura pensada antes de uma linha escrita. A escolha do banco de dados, do protocolo, da estrutura de dados — essas decisões definem se o sistema aguenta 10 ou 10 mil usuários.

05
DO HARDWARE AO SERVIDOR

Pensar na stack completa — do firmware rodando no microcontrolador até a API recebendo os dados na nuvem. Soluções IoT reais precisam de visão de ponta a ponta, não de silos.

06
APRENDIZADO CONTÍNUO

A tecnologia não para. Embedded systems, cloud, protocolos novos — ficar parado é regredir. Cada projeto é uma oportunidade de aprender algo que ainda não sei, e isso nunca vai mudar.
